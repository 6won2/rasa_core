This doesn’t need to be supported by every policy. If it is supported, the policy can be used for online training and the implementation for the continued training should be put into this function.

train(filename=None, max_history=3, augmentation_factor=20, max_training_samples=None, max_number_of_trackers=2000, remove_duplicates=True, **kwargs)[source]¶

Trains a policy on a domain using training data from a file.

Parameters:
  • augmentation_factor – how many stories should be created by randomly concatenating stories
  • filename – story file containing the training conversations
  • max_history – number of past actions to consider for the prediction of the next action
  • max_training_samples – specifies how many training samples to train on - None to use all examples
  • max_number_of_trackers – limits the tracker generation during story file parsing - None for unlimited
  • remove_duplicates – remove duplicates from the training set before training
  • kwargs – additional arguments passed to the underlying ML trainer (e.g. keras parameters)
Returns:

trained policy
